# Development Guide: releases

## Prerequisites

- **Git**
- **GitHub CLI** (`gh`)

## First-Time Setup

Clone the repository:

```bash
gh repo clone sixways-ai/releases
```

## Build

Not applicable. JSON files are auto-generated by GitHub Actions workflows.

## Run

Not applicable. The repository serves static files via GitHub Pages.

## Test

- Validate JSON files for correct structure and schema compliance.
- Test GitHub Actions workflows locally using [act](https://github.com/nektos/act):

```bash
act
```

## Debug

- Check GitHub Actions workflow run logs in the Actions tab of the repository.
- Use `gh run view` to inspect workflow runs from the command line.

## Common Tasks

| Task | Description |
|---|---|
| Publish a release | Create a GitHub Release with a version tag and attach build artifacts |
| Update the schema | Modify the JSON schema definition and validate existing files |
