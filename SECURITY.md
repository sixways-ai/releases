# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in SixWays, please report it responsibly.

**Do NOT open a public GitHub issue for security vulnerabilities.**

Instead, email **security@sixways.ai** with:

- Description of the vulnerability
- Steps to reproduce
- Impact assessment
- Any suggested fix

We will acknowledge receipt within 48 hours and provide a timeline for resolution.

## Supported Versions

| Version | Supported |
|---------|-----------|
| 0.1.x   | Yes       |

## Scope

This policy covers all SixWays open-source repositories:
- sixways-endpoint
- sixways-sandbox
- endpoint-vscode
- releases
